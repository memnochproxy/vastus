Notes on hextile maps

Apparently the pointy-top or flat-top orientation of hexes has various differences by heritage:

  • Pointy top -- used primarly in table top
  • Flat top -- used primarly in Battle-tech Some online map engines make this a configurable option. I can see how that wouldn't really change the overall aspect ratio of a map, both adjacent rows should still be offset by a pretty similar amount.

    Coordinates and Directions

    With all the possible orientations we might want to place tiles, I don't think we're going to cover all the directions. If we use clock directions (12) we lack a 45-deg orientation. Do we get that with 16 directions? If so, we can use hex. Should the encoding be Land.Degree? Where degrees are a choice of 16 positions (0-f: 0, 30, 45, 60, 90, 120, 135, 150, 180...) So a straight road headed east would look like: 

        O.0 O.0 O.0
    E.0 E.0 E.0
    O.0 O.0 O.0
    Removing the punctuation might be desirable: 
        O0 O0 O0
    E0 E0 E0
    O0 O0 O0
    Assuming some defaults will also be useful: 
        O O O
    E E E
    O O O
